iMedOne Flow

PMS Flow

1. Patient Management System (PMS) is maintenance and documentation of Patient’s administrative data in a Hospital Information System (HIS).

2. PMS is heart module of iMedOne.

Care Flow

1. The purpose of this module is to facilitate the planning and documentation of nursing care for Wards.

2. Care module also enables the generation of resource data that can be shared with the Govt. for further analysis and utilization.

LSM Flow

1. Leistung stelle  Management (LSM) is a module that handles all the services related to Radiology, Physiotherapy, and Psychiatry.

DOC.OP Flow

1. DOC.OP or Surgery documentation module allows to store and manage all kinds of data related to Docuobjects, such as the ones needed for the Operations.

Billing Flow

1. The billing module in hospital allows to manages the financial transactions related to the services provided by the hospital.

2. It allows the hospital staff to generate invoices, record payments, track outstanding balances, and generate reports.

AMS Flow

1. Ambulant Management System is a module that helps to manage the activities related to the outpatient department (OPD) of a hospital.

2. It allows the staff to register, schedule, and monitor the patients who visit the OPD for diagnosis, treatment, or consultation.

GRM Flow

1. The Global Resource Management (GRM) Planning module allows to create a comprehensive plan for patient's case, involving different entities as needed.
